Section Header

A heading that introduces a block of content within a page — a title, optional description, and optional section-level actions. For the title block at the top of a whole view, use the Page Header instead.

Anatomy

A title that names the block, with everything else optional: a short description, section-level actions, and a divider.

Team members

Manage your team members and their account permissions here.

1
Title — Names the block of content. The most common (and often only) part — keep it short, like a heading, not a sentence.
2
Description — Optional supporting line. Use it only when the block needs context the title can't carry.
3
Actions — Optional controls scoped to this section: a button, an icon (kebab) menu, or search & sort for the content below.
4
Divider — Optional hairline that separates the heading from the content. Skip it when cards or spacing already group the block.

Best practices

In practice most section headers are just a title — add the rest only when the block needs it.

→
Title-led. A short, scannable title is enough for most sections. Reach for a description, actions or a divider only when they earn their place.
→
It's a sub-heading, not the page title. Use it to divide a page into blocks; the view as a whole is named by the Page Header.
→
Keep actions section-scoped. The controls on the right act on this block's content — "Add", a kebab menu, or search & sort — not on the whole page.
→
Actions sit to the right. Place the section's actions to the right of the title and description, aligned with the heading.
→
Divider is optional. Add it to separate the heading from dense content; leave it off when the block already sits in a card or has clear spacing.

Design Tokens

Type and spacing on the page surface — the only fill is the optional divider.

TokenRoleValue
--content-baseTitle text#192743
--content-secondaryDescription textrgba(25,39,67,.8)
--border-baseDivider#e2e4e9
--border-actionAction button / search borders#bcc1cd

Typography: title text-l4 (18/SemiBold), description body-14 (14/Regular). Spacing: 4px between title and description, 16px above the divider, 24px between the text and the actions.

Accessibility

A section header is a heading — its structure has to read correctly for assistive tech.

→
Use a real heading level. Render the title as h2/h3 nested under the page's h1 — don't skip levels and don't fake a heading with bold text.
→
Associate actions with the section. The controls belong to the block this header introduces; keep them next to it so the relationship is clear.
→
Name icon-only actions. A kebab/options icon button needs an accessible name ("More", "Options").
→
Reuse component semantics. Search, sort and menus in the header follow their own components' roles and keyboard behaviour.
