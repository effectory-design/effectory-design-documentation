List

A vertical stack of rows separated by hairline dividers. Each row pairs a leading content block with optional trailing meta, a status tag or an action — the pattern behind survey lists, project lists and settings screens.

Anatomy

A container that stacks rows. Each row has a leading content block that flexes to fill the width, with any trailing content pinned to the right.

Content — The leading block: a title plus an optional sub-line (a path, meta, a timestamp). It flexes and truncates so long text never pushes the trailing content off-screen.
4
Trailing — Right-aligned meta, a status tag, or an action. Anything that describes or acts on the row.
5
Divider — A hairline between rows; the last row has none.

Accessibility

Give the list real list semantics and make interactive rows honest targets.

→
List semantics. Back the list with a <ul>/<li> (or an appropriate ARIA role) so assistive tech announces “list, N items”.
→
One target per row. A clickable row is a single <a> or <button>, not a div with a handler — one focus stop, one announcement. Nested actions must not sit inside the row-level link.
→
Visible focus. Interactive rows need a clear focus ring, not just the hover fill (hover isn't available to keyboard users).
→
Meaning beyond colour. A trailing status tag carries meaning in its text, not only its colour — see the Tag page.
→
Empty state. When filtering empties the list, show a short message (.list-empty) rather than a blank panel.

Row with actions (buttons + kebab)

Group trailing actions in a .list-item-actions cluster (tight 8px gap) so they read as one unit. A row that carries its own buttons is a static .list-item — not .is-interactive: you can't nest buttons inside a row-level <a>. If the row also navigates, make the title itself the link.

Empty state

When a filter empties the list, render a .list-empty message in place of the rows.

<div class="list-empty">No surveys match your filters.</div>
⚠️
Figma / styleguide gap These .list / .list-item classes are the prototype pattern extracted from the surveys screen. Confirm the row anatomy, the divider treatment and the interactive-row markup (single <a> vs a wrapping element) against the Angular styleguide before standardising. The trailing chevron is a documentation affordance — production rows may use a kebab menu or nothing.

CSS classes

Prototype classes for the list (components.css).

ClassRole
.listThe container. Stacks its rows in a column.
.list-itemOne row. Leading content + trailing nodes, with a bottom divider.
.list-item.is-interactiveMakes the whole row clickable; adds a hover fill and pointer.
.list-item-contentThe leading block that flexes to fill the width.
.list-item-titlePrimary line (16px SemiBold, truncates).
.list-item-subSecondary line (12px) — meta, a path, a timestamp. Add a 12px <i data-icon> to lead it.
.list-item-trailingOptional wrapper to group several trailing nodes. A single trailing node can sit directly in the row.
.list-item-actionsTight cluster (8px gap) for trailing buttons and/or a kebab. Use on a static row — not .is-interactive.
.list-emptyCentered message shown when the list has no rows.
