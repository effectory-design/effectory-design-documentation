Built for two worlds
Designers and developers, united
One system, two entry points. Everything syncs — from the Figma variable to the CSS custom property.
For Designers
Work directly in Figma with components that map 1:1 to production. Every variant, every state, every token — exactly as it ships.
Figma variables bound to design tokens
Anatomy diagrams and usage guidance per component
Light, dark, Coordinator and Personal themes

Angular components with typed inputs and outputs
Copy-ready code snippets per example

Get started
Pick your path
Three common ways to get up to speed, depending on where you're starting from.
1
Understand the foundations
Start with Colors, Typography, and Spacing. These three token layers underpin every component in the system. Understanding them makes everything else click.
2
Explore components by category
Each component page has two tabs: Design (anatomy, usage guidelines, states, tokens) and Developer (live playground, API table, code snippets). Browse the category cards above or use the sidebar.
3
Read the Design Principles
The Design Principles explain the decisions behind the system — why things are the way they are — and serve as the tiebreaker when two valid options exist.
