Announcement Dialog

A centred, modal release tour. It walks people through several new features one screen at a time — a festive intro, a coloured step per feature, and a closing screen. For a small, non-blocking card in the corner, see the Announcement Card.

Anatomy

A full-screen modal: a coloured illustration up top, a story-style progress bar, a title and body, and a footer that moves the tour forward.

A tour runs from a festive intro — a livelier illustration with a confetti-and-sparkle flourish — through one feature step per release, to an outro that fades into the surface and links to the full release notes. Each screen automatically takes the next colour from the palette, so the tour moves through colour as people advance; the colours are decorative and carry no status meaning.

Accessibility

Unlike the Card, this is a modal — it takes over the screen, so it must manage focus carefully.

→
Trap focus while open. Use role="dialog" with aria-modal="true" and keep keyboard focus inside the tour until it closes.
→
Easy to dismiss. Escape closes it, the backdrop is clickable, and the close button is always reachable — the tour is optional, never a trap.
→
Return focus. When it closes, move focus back to the control that opened it (or a sensible place in the page).
→
Name the controls. The close and back icon buttons are icon-only — give them accessible names ("Close", "Previous").
→
Progress isn't colour-only. Convey the step position to assistive tech (e.g. "Step 2 of 4"), not just through the filled segments.
→
Respect reduced motion. The intro burst, confetti and slide animations should fall back to a simple fade under prefers-reduced-motion.
→
Illustrations are decorative. They carry no information, so hide them from assistive tech (empty alt / aria-hidden).

Accessibility

Implement it as a focus-trapping modal that's always easy to leave.

→
role + aria-modal. Render with role="dialog" and aria-modal="true", labelled by the step title.
→
Focus management. Trap focus while open, close on Escape and backdrop click, and return focus to the trigger on close.
→
Announce progress. Expose "Step N of M" to assistive tech — don't rely on the coloured segments alone.
→
Named icon buttons + reduced motion. Give the close/back buttons accessible names, and fall back to a fade when prefers-reduced-motion is set.
