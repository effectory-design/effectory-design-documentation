Grid & Layout

Engage doesn't use a strict column grid. Layout is shaped by three things: a responsive page padding, a content container width, and a consistent gutter between elements.

Approach

There is no formal 12-column grid. Compose layouts from these primitives instead:

→
Page padding. A responsive margin around the content that grows with the viewport — 16px on mobile up to 64px on desktop.
→
Container width. A max-width on the content: narrow, wide, or full — pick the one that fits the content type.
→
Gutter. The space between elements — 24px by default. Keep it consistent within a view.

Breakpoints

Three ranges drive the responsive page padding and the navigation mode. The navigation switches between the sidebar and a header (hamburger) at 1200px.

Container widths

The content sits in a centred container. Choose the width that matches the content — not every screen should be full-bleed.

Gutter

Use a 24px gutter between elements by default — cards in a row, columns, stacked blocks. Keep one consistent value within a view rather than mixing gaps.

App shell

On desktop (≥1200px) a 240px sidebar is always visible and the content takes the remaining width. Below 1200px the sidebar collapses into a header with a hamburger menu, and the content goes full width.
